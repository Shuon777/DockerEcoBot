EcoBot Core API — /search_pipeline
====================================

URL: POST http://<host>/core-api/search_pipeline
Content-Type: application/json; charset=utf-8


ТЕЛО ЗАПРОСА
------------
{
  "query":   "строка"   // обязательно — текстовый запрос пользователя
  "user_id": "строка"   // опционально — включает контекст диалога между запросами
}

Пример:
  {"query": "Как выглядит эдельвейс"}


ПРИМЕР ОТВЕТА
-------------
{
  "slots": {
    "object_type": "Объект флоры и фауны",   // или "Географический объект" / "Услуга"
    "synonym":     "эдельвейс",              // конкретное название, null если не определено
    "modality":    "Изображение",            // "Текст" / "Изображение" / "Геоданные"
    "features":    {},                       // фильтры: {"Время года": "Лето", ...}
    "properties":  {},                       // {"Детальное расположение": "Иркутск", ...}
    "template":    "О4",
    "modality_ambiguous": false
  },

  "result": {
    "answer": "Эдельвейс имеет красивые цветки...",  // текстовый ответ (если есть)
    "images": ["large.jpg", "large2.jpeg"],           // список файлов (если modality=Изображение)
    "map":    {"static": "url", "interactive": "url"} // карта (если modality=Геоданные)
  },

  "proactive": {
    "photo": "эдельвейс",   // что предложить следующим (опционально)
    "map":   "эдельвейс"
  },

  "is_continuation":   false,  // true если слоты унаследованы из предыдущего хода
  "modality_ambiguous": false,
  "simplifications":   []      // варианты с ослабленными фильтрами если результат пустой
}


ВАЖНО
-----
- Кириллицу передавать строго в UTF-8 (curl на Windows кодирует неверно — использовать Python/Postman)
- Если передан user_id — контекст сохраняется в Redis, слоты наследуются между запросами
