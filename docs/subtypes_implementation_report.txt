        в текущие (AdminPanel/templates/)


1. ЦЕЛЬ РАБОТЫ
--------------

Перенос реализации подтипов (subtypes) из эталонных шаблонов (AdminPanelTemplates/)
в текущие рабочие шаблоны (AdminPanel/templates/). Подтипы - это множественное
свойство объекта, которое реализовано через интерактивные chips-теги с
автодополнением из справочника (datalist).


2. АРХИТЕКТУРА РЕШЕНИЯ
-----------------------

Каждый объект (биологический, географический, услуга) может иметь несколько
подтипов. Подтипы хранятся как массив строк в
object_properties['Подтип объекта']. Реализация состоит из трёх уровней:

- HTML-разметка: контейнер chips (#subtypes-chips), datalist для автодополнения,
  скрытое поле (#subtypesHidden) для отправки данных формы, input с привязкой
  к datalist.

- JavaScript-функции:
  * addSubtypeChip() - добавление чипа с валидацией через datalist
  * removeSubtypeChip() - удаление чипа
  * onSubtypeInput() - авто-добавление при выборе из datalist
  * syncSubtypesHidden() - синхронизация скрытого поля (только для форм создания)

- Серверная часть: сохранение подтипов как массива строк через
  object_properties['Подтип объекта'].

  БЫЛО:

    <datalist id="subtypesList">...</datalist>
    <div class="form-text text-muted mt-1">

    </div>

  СТАЛО:

5. СОХРАНЁННАЯ ФУНКЦИОНАЛЬНОСТЬ
--------------------------------

При переносе изменений из эталонных шаблонов была сохранена функциональность,
которая отсутствует в эталонных шаблонах:

  - Модалка editSynonymsModal с SortableJS для управления синонимами
  - Интеграция с серверным API через saveProperties()
  - Сбор подтипов в объект props['Подтип объекта'] при сохранении


6. ЗАКЛЮЧЕНИЕ
-------------

Все 6 шаблонов успешно синхронизированы с эталонной реализацией подтипов.
Текущие шаблоны теперь используют chips-интерфейс с валидацией через datalist,
что обеспечивает:

  [V] Удобный множественный выбор подтипов через интерактивные теги
  [V] Валидацию значений - только из справочника (datalist)
  [V] Защиту от дублирования подтипов
  [V] Автоматическое добавление чипа при выборе из выпадающего списка
  [V] Корректную отправку данных через скрытое поле (для форм создания)
  [V] Совместимость с существующей серверной логикой
